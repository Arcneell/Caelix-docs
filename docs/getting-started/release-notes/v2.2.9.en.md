# Release notes — v2.2.9

**Date**: July 2026
**Image**: `ghcr.io/arcneell/caelix:2.2.9` (also `:latest`)

> Robustness batch from the latest audit: anti-flood for failed backups, the
> anti-race guard on the remaining views, and hardening of the HA loop / autoscaler.

---

## Robustness — backups

A persistently failing backup never updates its success timestamp, so it stays "due"
on every reconcile pass (~15s) and re-emits a `backup_failed` incident each time (the
2.2.8 flood). A **failure backoff** now spaces out retries (default 15 min, tunable per
app via `backup_fail_backoff`, capped at the backup interval). A success clears the
backoff and the normal cadence resumes.

The dump-tool selection (`mariadb-dump` else `mysqldump`) added in 2.2.8 is now covered
by a bats test, alongside tests for the backoff.

## Robustness — frontend (anti-race guard)

The `useRequestGuard` composable (monotonic `reqId` anti-race guard, added in 2.2.5) now
covers the remaining polled views: **Services**, **Logs**, **Autoscale**, **Domains**,
**Certificates**, **Stacks** and **App-Stack detail**. A slow response (poll or fan-out)
can no longer clobber fresher data from a manual refresh, a scope switch, or an
action-triggered refetch.

## Robustness — cluster HA & autoscaler

- **Leadership lease**: if minting a new lease session fails (store unreachable /
  partition), the controller returns an empty session instead of reusing the old,
  now-expired id — acquiring leadership under a dead lease becomes a clean no-op, and the
  next pass retries once the store is back.
- **Autoscaler (HPA)**: when rewriting the manifest, HPA now applies its **delta**
  (`+1` / `-1`) to the `total_replicas` value **re-read** right before the write, then
  re-clamps — instead of writing an absolute value derived from a possibly stale read.
  A concurrent admin edit to the same service is no longer clobbered.

## Misc

- `pydantic` pinned (`==2.13.4`) alongside the other backend dependencies, for
  reproducible builds.

## Validation

Full suite (pytest + bats + shell checks + frontend build) green, then the `2.2.9`
image was deployed to the live 3-node cluster with quorum, floating VIP and etcd auth
verified.
