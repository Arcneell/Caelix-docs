# Release notes — v2.2.15

**Date**: July 2026
**Image**: `ghcr.io/arcneell/caelix:2.2.15` (also `:latest`)

> From the continuous audit campaign: closes a **secret-disclosure** to the technicien role,
> an anti-crash guard on `auto_update_interval`, ends OOMs on large exports/backups, and
> hardens the frontend/cluster.

---

## Security — secret disclosure to technicians (HIGH)

Three reads returned the manifest / `.env` **in plaintext** (backup FTP/rsync passwords,
service `env` blocks) to any authenticated user, even though `/state` already scrubs them for
non-admins and their write counterparts are admin-only:
`GET /api/manifest/show`, `GET /api/manifest/raw`, `GET /api/stacks/{name}/env`.
They are now **`require_admin`**.

## Engine — anti-crash guard + robustness

- **`auto_update_interval`** with a non-numeric value (e.g. `nightly`) reached `(( ))` → a fatal
  `set -u` error that truncated the reconcile pass. Now coerced at runtime and rejected by
  `caelix validate` (the `hourly`/`daily`/`weekly` aliases are still accepted).
- **etcd credentials JSON-escaped** in the auth body (`lib/node.sh`): a password containing
  `"`/`\` no longer breaks the handshake.

## Backend — ends OOM on large transfers

`docker save` (image export), `docker export` (container export) and the config backup buffered
the **entire archive in RAM** → OOM risk on a multi-GB image/container. They now write to a temp
file and **stream it from disk** (`FileResponse`, cleaned up after send). Bounds added on
`limit` (`/incidents`, `/audit`); two catch-all `except (X, Exception)` narrowed.

## Frontend — stable keys (no more mis-bind)

`splice`-removable editor lists used the array index as `:key` → deleting a middle row jumped
field values/focus to the wrong row. Now keyed on a **stable identity** (`_id`):
CreateContainerView (ports/volumes/env/labels + capabilities/devices/ulimits/dns/hosts/tmpfs/
log-opts/sysctls lists), ServicesView and StackDetailView (env vars). ContainerDetailView labels
routed through `t()`.

## Cluster (HA)

- **`renew_session` retries the keepalive once** before declaring the lease lost: a transient
  etcd 5xx no longer revokes a still-valid lease (leadership churn / VIP flap on a passing
  error). A real partition still fails after the retry.

## Validation

Full suite (744 pytest + 304 bats + shell checks + frontend build) green, then the `2.2.15`
image was deployed to the live 3-node cluster with etcd quorum, floating VIP and etcd auth
verified.
