# Release notes — v2.2.20

**Date**: July 2026
**Image**: `ghcr.io/arcneell/caelix:2.2.20` (also `:latest`)

> Two missed surfaces of the "technicians see no secrets" boundary, plus three blocking etcd
> handlers — found by the convergence audit.

---

## Security — the state stream no longer leaks secrets to technicians (HIGH)

The SSE stream **`GET /api/stream`** returned the **full unsanitized state** to any
authenticated user, while its REST sibling `GET /api/state` already applies
`sanitize_state_for_role`. A technician therefore received notify credentials (SMTP/webhook)
and per-service `env`/`volumes_bind` that `/state` strips. The stream now applies the **same
role-based scrub** using the SSE ticket's role.

Same class closed on two other reads: **`GET /containers/{name}/inspect`** and
**`/clone-config`** now redact `Config.Env` / `env` for non-admins (raw `docker inspect` is no
longer a boundary bypass).

## Backend — three blocking etcd handlers moved off the event loop

`manifest_raw_get`, `list_webhooks` (reads) and `templates_save` (write) did synchronous
`read_text`/`write_text` I/O — blocking etcd round-trips in cluster mode. Moved to a sync
handler (threadpool) or `asyncio.to_thread`.

## Validation

Full suite (748 pytest + 306 bats + shell checks + frontend build) green, then `2.2.20`
deployed to the live 3-node cluster (backend-only: etcd 3/3 quorum, floating VIP, etcd auth
verified).
