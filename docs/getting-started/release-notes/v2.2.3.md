# Notes de release — v2.2.3

**Date** : juillet 2026
**Image** : `ghcr.io/arcneell/caelix:2.2.3` (également `:latest`)

> Correctifs de **qualité, correction et robustesse** issus d'un audit global du code
> (au-delà de la sécurité). Aucun changement fonctionnel visible.

---

## Backend (correction / performance)

- **Endpoints cluster non bloquants** : les handlers de `cluster.py` (et `metrics`,
  `logs`, `incidents`, `registry`) exécutaient des I/O synchrones (etcd, docker,
  fichiers) **sur l'event-loop**, gelant toutes les requêtes et flux SSE. Ils tournent
  désormais dans le threadpool.
- **Fuites mémoire des limiteurs** : les compteurs anti-brute-force (rate-limit,
  lockout login) ne libéraient jamais les clés expirées → croissance non bornée sous
  rotation d'IP. Balayage opportuniste + suppression des clés vides ajoutés.
- **Courses d'écriture** : `certificates.json` (renouvellement vs upload) sous un lock
  partagé ; acquittement d'incidents sérialisé ; HPA applique ses changements par
  merge (re-lecture) au lieu d'écraser le manifeste ; cache d'état verrouillé.
- **Robustesse** : store utilisateurs cluster corrompu → erreur contrôlée (plus de
  500 sur tout login) ; historique de notifications non perdu sur erreur de lecture
  transitoire ; échec de livraison de notification journalisé au lieu d'être avalé ;
  `registry` tolère une entrée sans `name`.

## Cluster / HA

- **Autoscaler HPA** mesure désormais **toutes** les répliques (`-rN`), pas seulement
  la première → plus de scale-down prématuré / oscillation ; compteurs purgés pour les
  apps supprimées.
- **Hystérésis de résignation** : le contrôleur ne cède plus le leadership (et la VIP)
  sur un seul heartbeat manqué — il exige 2 ticks consécutifs (anti-flapping).

## Moteur shell

- **Un fichier timestamp corrompu ne tue plus la passe de réconciliation** (garde
  numérique avant arithmétique sous `set -u`, dans backup et update).
- **Backup DB** vérifie le code de sortie du dump (`PIPESTATUS`) et l'intégrité gzip
  (`gzip -t`) — un dump tronqué n'est plus enregistré comme réussi (et est supprimé).
- **Proxy** : `Content-Length` non numérique et `--health-interval` invalide gérés
  (plus de handler qui casse ni de busy-spin 100% CPU) ; id de leader etcd nettoyé
  (bind VIP fiable) ; métrique autoscale validée avant comparaison.

## Frontend / build

- Cap du buffer de logs live (fuite mémoire sur flux long).
- Image release buildée sur **Node 20 (LTS)** — la toolchain validée par la CI
  (auparavant Node 26, jamais testé par la CI).

## Tests

- Chiffrement at-rest exercé pour de vrai (seal/open via `shared_state`, migration) ;
  test du retry 401 etcd ; cost bcrypt configurable (suite ~2× plus rapide en CI).
