# Release notes — v2.2.18

**Date**: July 2026
**Image**: `ghcr.io/arcneell/caelix:2.2.18` (also `:latest`)

> Audit follow-ups: the 2.2.17 removal tombstone was cleared under an **unsanitized** id
> (re-added node stuck), the fast heartbeat loop didn't honor it, and two engine `sleep`s
> were still unguarded.

---

## Cluster — tombstone (follow-up to 2.2.17)

- **Re-join clear fixed.** The agent writes/reads the tombstone under the **sanitized** id
  (`_node_sanitize_id`), but `install.sh` cleared it under the **raw** `--node-id` → a re-added
  node stayed self-fenced until the lease expired (24h). `install.sh` now applies the same
  sanitization.
- **Fast heartbeat loop** now honors the tombstone (like the fence): a removed leader no longer
  re-binds the VIP during the window before the main loop fences it.

## Engine

- `candidate_grace` / `post_repair_grace` (`lib/repair.sh`) reached `sleep` unguarded → a
  `sleep infinity` would have hung the repair path. Now `^[0-9]+$`-guarded (last unguarded
  manifest values).

## Frontend

- Two hardcoded labels (`Route`, `Template`) routed through `t()` (+ fr/en keys).

## Validation

Full suite (747 pytest + 306 bats + shell checks + frontend build) green, then `2.2.18`
deployed to the live 3-node cluster (etcd 3/3 quorum, floating VIP, etcd auth verified; all 3
nodes keep their `/meta`, no false fencing).
