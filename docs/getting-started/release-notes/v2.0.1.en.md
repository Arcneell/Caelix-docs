# Release notes — v2.0.1

**Date**: June 2026  
**Image**: `ghcr.io/arcneell/caelix:2.0.1` (also `:latest`)

> A **bug-fix** release on top of 2.0, from an end-to-end testing pass on a real
> 3-node cluster and on a single host. It makes cluster operations (scale-down,
> service removal, failover) **converge reliably**, and fixes several single-host
> issues. No breaking changes.

---

## Cluster fixes (the important ones)

- **Scale-down and removal finally converge.** A node taken out of the plan kept its
  stale sub-manifest and never removed the scaled-down/removed replicas. The agent now
  also adopts a **present-but-empty** sub-manifest (only locally pinned sections —
  console, proxy — survive), so a lowered `total_replicas`, an HPA scaling down, or a
  removed service **propagate and clean up on their own**, with no agent restart.
- **No more phantom ingress backends.** A removed replica's backend stayed registered
  and VIP:80 kept routing to a gone replica. The agent now **deregisters** backends for
  apps no longer in its manifest.
- **No more port conflict on failover.** An ingress service with a **fixed host port**
  is now capped at **one replica per node**: on node loss the excess replica goes
  *pending* instead of looping on `port is already allocated`.
- **`remove_orphans` defaults ON for every cluster node**: the controller's plan is
  authoritative, so any `caelix-*` container not in the plan is reaped (an explicit
  value in `[orchestrator]` still wins).

## Single-host fixes

- **Console-triggered volume backup**: a bind mount under the Caelix root failed
  (`no_existing_paths`) because the backend stat'ed the **host** path from inside the
  container. Paths are now translated to their in-container location before archiving.
- **Initial admin password**: it is now also written to `.caelix/initial-admin-password`
  (mode 0600), surfaced by the installer, and removed on the first password change — no
  longer lost when the console container is recreated during install.
- **`caelix status`** no longer depends on the current directory (`./.caelix`): it reads
  state from the install root, so no more false "daemon not running".
- **Docs**: the documented default login (`admin`/`admin`) is corrected — the password is
  **random** on first start.

---

## Upgrade

```bash
docker run --rm ghcr.io/arcneell/caelix:latest cat /opt/caelix/install.sh | bash -s -- --with-systemd
```

On a cluster, update each node (image + engine re-extraction) then let the agent
reconcile. No configuration changes required.
