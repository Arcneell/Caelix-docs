# Release notes — v2.2.22

**Date**: July 2026
**Image**: `ghcr.io/arcneell/caelix:2.2.22` (also `:latest`)

> The last stragglers from the exhaustive audit (all ~80 GET endpoints enumerated): one
> template secret leak and five blocking etcd I/Os.

---

## Security — custom template bodies hidden from technicians (HIGH)

`GET /api/templates/custom` returned the **raw body** of every custom template — compose/manifest
that can embed `env`, `volumes_bind` and inline credentials — to any authenticated user.
Non-admins now get the **metadata** (title, description) but not `content`. Same boundary as
`/state`, `manifest/raw`, the app-stack detail view, `inspect` and stack compose.

## Backend — five blocking etcd I/Os moved off the event loop

Five `async` handlers did synchronous `read_text`/`write_text`/`dir_*` I/O (blocking etcd
round-trips in cluster mode) directly on the event loop: `get_stack_env`, `save_stack_env`,
`get_sources`, and the exists/delete checks in `update_custom_template` / `delete_custom_template`.
All moved to `asyncio.to_thread`.

With this, the technician-no-secrets boundary and the blocking-on-loop class are **fully swept**
(exhaustive audit of all ~80 GET endpoints + every async handler).

## Validation

Full suite (751 pytest + 306 bats + shell checks + frontend build) green, then `2.2.22`
deployed to the live 3-node cluster (backend-only).
