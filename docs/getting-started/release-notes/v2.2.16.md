# Notes de release — v2.2.16

**Date** : juillet 2026
**Image** : `ghcr.io/arcneell/caelix:2.2.16` (également `:latest`)

> Lot d'audit continu : fermeture d'une **fuite de secrets** au rôle technicien, durcissement
> de plusieurs entrées admin (injection de flag / traversal / bind), fin des OOM sur imports
> volumineux, dernière garde arithmétique moteur, et clés stables sur un composant partagé.

---

## Sécurité

- **HIGH — fuite de secrets aux techniciens.** `GET /api/app-stacks/{name}` renvoyait le bloc
  `env` (mots de passe DB, clés d'API) et la `config` complète de chaque service à tout
  utilisateur authentifié. La vue détaillée est désormais **réservée aux admins** (les
  non-admins reçoivent la liste assainie, comme sur `/state`).
- **Durcissement (admin) :** `POST /api/networks/create` rejette les valeurs `driver/subnet/
  gateway` préfixées par `-` (injection de flag docker) et valide le nom ; `target_service`
  (domaines) validé comme nom docker (empêche un `../…` d'écrire hors de son dossier) ;
  déploiement de template valide `image`/`name` (injection de flag) ; ajout de service
  d'app-stack applique `_validate_stack_bind_path` (plus de bind d'un chemin hôte arbitraire),
  cohérent avec le chemin de déploiement.

## Backend — fin des OOM sur uploads

`import` d'image (2 Go) et `restore` de backup (500 Mo) bufferisaient tout le corps de requête
en RAM avant écriture. Ils **streament désormais** la requête vers un fichier temporaire par
morceaux, avec plafond appliqué **au fil de l'eau** (rejet 413 dès dépassement) — plus de pic
mémoire pouvant OOM-killer la console.

## Moteur

- `autoscale_up_threshold` re-lu sur le chemin « métrique critique » (`autoscale.sh`) était la
  dernière lecture non gardée → garde `^[0-9]+$` ajoutée (plus d'abandon de passe autoscale).

## Frontend

- `ArrayField` (composant partagé des assistants conteneur/service) utilisait la clé d'index
  sur des lignes supprimables → suppression d'une ligne du milieu mal rebindée. Passé à une
  **clé d'identité stable** (WeakMap sur l'objet-ligne), sans changement de payload.

## Validation

Suite complète (pytest + bats + vérifications shell + build frontend) au vert, puis
déploiement de l'image `2.2.16` sur le cluster 3 nœuds live (quorum etcd, VIP flottante,
auth etcd vérifiés).
