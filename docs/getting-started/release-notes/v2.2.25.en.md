## Backend — user-store access moved off the event loop (HIGH in cluster)

In cluster mode the user store lives in etcd: every `get_user_by_username` / `get_user_by_id` /
`list_users` / `create_user` / `update_user` / `delete_user` is a **blocking** network round-trip
(synchronous `urllib`). Every `async` handler in `routers/auth.py` called them directly on the
event loop, freezing **all** requests/SSE for each round-trip — `login` and `me` being the hottest
paths (every login attempt).

Each call now goes through `asyncio.to_thread`: `login`, `me`, `change_password` (3 calls),
`get_users`, `create_new_user`, `update_existing_user` (3 calls), `delete_existing_user` (2 calls).
bcrypt hashing was already offloaded; only the store access was exposed. With this, the
blocking-on-loop class is **fully swept** (routers + auth dependencies + background tasks +
middlewares).

## Cluster — orphan tombstone lease avoided (LOW)

Follow-up to the 2.2.24 `remove_node` fix: on the error path (retry after a failed tombstone
write), the freshly granted etcd lease was not revoked → up to ~2 orphan leases (24 h TTL,
self-expiring) per failed removal. `remove_node` now revokes the lease before retrying.
