# Release notes — v2.2.5

**Date**: July 2026
**Image**: `ghcr.io/arcneell/caelix:2.2.5` (also `:latest`)

> Fixes data races in the console (frontend). No visible behaviour change under normal
> operation.

---

## No more stale data on screen

The auto-refreshing views (containers, images, networks, volumes, nodes, cluster
overview, dashboard, app-stacks…) fired a fetch every ~8–10 s on top of manual
refreshes, scope/filter switches and post-action refetches. Because a multi-node
fan-out can be slower than the interval, an **older** response could land **after** a
newer one and **overwrite** the fresh data (or another scope's rows) — a classic
last-write-wins race.

- New `useRequestGuard` composable (monotonic request id): each fetch is tagged and its
  result is applied only if it is **still the latest**; starting a new fetch (poll tick,
  scope switch, filter toggle) immediately invalidates the previous in-flight response.
- Applied to every polled fan-out listing: Containers, Images, Networks, Volumes,
  Nodes, Overview, Dashboard, App-Stacks, Node detail.
- Container detail: the one-shot stats fetch no longer overwrites the **live** SSE
  stream when it is already active.

## Validation

`vue-tsc` + ESLint + Vitest green (added a unit test for `useRequestGuard`); console
rebuilt and served on the 3-node cluster (load + navigation OK).
