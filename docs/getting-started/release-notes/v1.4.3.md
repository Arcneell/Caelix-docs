# Notes de release — v1.4.3

**Date** : juin 2026  
**Image** : `ghcr.io/arcneell/caelix:1.4.3` (également `:latest`)

---

## Résumé

Correctif de **durcissement de l'authentification** : la révocation des tokens lors d'un changement de mot de passe est désormais fiable.

## Changement

- Les horodatages `created_at` / `updated_at` des utilisateurs passent à une **précision microseconde**.
  
  La révocation des sessions compare le claim `pw_ts` du token à `updated_at` (égalité de chaîne). Avec l'ancienne précision **à la seconde**, un token émis dans la **même seconde** qu'un changement de mot de passe n'était pas invalidé (les deux `updated_at` étant identiques). La précision microseconde garantit que tout changement de mot de passe produit un `updated_at` distinct, donc l'invalidation des anciens tokens est systématique.

Impact réel faible (fenêtre d'une seconde), mais c'est la bonne garantie de sécurité — et ça supprime un test d'authentification intermittent.
