# Release Notes — v1.4.3

**Date**: June 2026  
**Image**: `ghcr.io/arcneell/caelix:1.4.3` (also `:latest`)

---

## Summary

**Authentication hardening** fix: token revocation on password change is now reliable.

## Change

- User `created_at` / `updated_at` timestamps now use **microsecond precision**.

  Session revocation compares the token's `pw_ts` claim against `updated_at` (string equality). With the previous **second** precision, a token issued in the **same second** as a password change was not invalidated (both `updated_at` values were identical). Microsecond precision guarantees that any password change produces a distinct `updated_at`, so old tokens are always revoked.

Low real-world impact (one-second window), but it's the correct security guarantee — and it removes an intermittent authentication test failure.
