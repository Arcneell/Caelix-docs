# Release notes — v2.2.19

**Date**: July 2026
**Image**: `ghcr.io/arcneell/caelix:2.2.19` (also `:latest`)

> Two backend fixes from the convergence audit: a cluster-manifest secret leak and blocking
> etcd calls in the request middleware.

---

## Security — cluster manifest is admin-only (HIGH)

`GET /api/cluster/manifest` was `require_auth` and returned the **raw cluster manifest**
(service `env` secrets, `volumes_bind`, backup transport credentials) to any authenticated
user — the same leak class as `/manifest/raw`, `/stacks/{name}/env` and the app-stack detailed
view, already closed. The write (`PUT`) was already admin; the read is now **`require_admin`**.

## Backend — no more blocking etcd calls in the middleware (HIGH)

`NodeTargetMiddleware` runs on **every request** and resolved the target Docker daemon
(`resolve_node_docker_host` / `resolve_container_node_docker_host`) **synchronously** — in etcd
cluster mode those are blocking HTTP round-trips (and, for container auto-targeting, a loop over
nodes). Each request stalled the event loop behind etcd latency. The resolution now runs via
`asyncio.to_thread` (matching the rest of the codebase).

## Validation

Full suite (747 pytest + 306 bats + shell checks + frontend build) green, then `2.2.19`
deployed to the live 3-node cluster (backend-only: etcd 3/3 quorum, floating VIP, etcd auth
verified).
