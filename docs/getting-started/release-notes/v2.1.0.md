# Notes de release — v2.1.0

**Date** : juin 2026  
**Image** : `ghcr.io/arcneell/caelix:2.1.0` (également `:latest`)

> La 2.1 remplace le store de cluster par **etcd (Apache-2.0)** et retire entièrement
> Consul, protège les conteneurs d'infrastructure depuis la console, et modernise
> l'interface (thème en dégradé, squelettes de chargement). Aucune rupture de
> configuration pour les services ; le mono-hôte est inchangé.

---

## Store de cluster : etcd (Apache-2.0), Consul retiré

- Le plan de contrôle HA repose désormais sur **etcd** : chaque nœud exécute un
  membre etcd (quorum Raft) et un controller, l'élection du leader se fait par
  **lease + transaction « créer si absent »**, et l'état partagé (utilisateurs,
  secret JWT, configuration, stacks, certificats) vit dans etcd. Le client est sur
  `:2379`, le raft sur `:2380`.
- **Consul est entièrement supprimé** du produit (code, options, documentation).
  L'installeur ne propose plus que le store etcd ; un nœud rejoint le cluster avec
  `--mode join --store-addr http://<nœud-existant>:2379` (l'ajout au cluster etcd est
  automatique via `etcdctl member add`).
- Conséquence pour la distribution commerciale : **plus aucune dépendance sous
  licence BSL**. Un déploiement standard n'embarque que des composants permissifs.

!!! note "Migration depuis la 2.0"
    Le store change de Consul vers etcd. Sur chaque nœud : mettez à jour l'image et
    ré-extrayez le moteur, retirez l'ancien conteneur Consul (`docker rm -f
    caelix-consul` + le volume), puis réinstallez en mode cluster avec `--store-addr`.
    Le controller en premier, les autres nœuds ensuite.

## Protection des conteneurs d'infrastructure

La console marque maintenant ses conteneurs d'infrastructure — la console web
(`caelix-caelix-ui`) et le store (`caelix-etcd`) — comme **système**. Ils sont
**masqués par défaut** dans la vue Conteneurs (bouton « Système » pour les afficher),
signalés par un badge, et tout arrêt ou suppression passe par un **avertissement
explicite** : l'opération casse le nœud et peut casser le quorum du cluster.

## Interface

- **Thème en dégradé** appliqué à toute la console, en variantes sombre et claire,
  avec une barre latérale et un en-tête translucides. Le thème clair n'est plus un
  aplat blanc.
- **Squelettes de chargement** : les listes de ressources (conteneurs, images,
  volumes, réseaux, services, stacks…) affichent un gabarit animé pendant le
  chargement au lieu d'apparaître d'un coup.

## Mise à jour

```bash
docker run --rm ghcr.io/arcneell/caelix:latest cat /opt/caelix/install.sh | bash -s -- --with-systemd
```

En cluster, mettez à jour chaque nœud (image + ré-extraction du moteur) ; voir la
note de migration ci-dessus pour le passage de Consul à etcd.
