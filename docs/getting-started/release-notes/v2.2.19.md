# Notes de release — v2.2.19

**Date** : juillet 2026
**Image** : `ghcr.io/arcneell/caelix:2.2.19` (également `:latest`)

> Deux correctifs backend de l'audit de convergence : une fuite de secrets du manifeste
> cluster et des appels etcd bloquants dans le middleware.

---

## Sécurité — manifeste cluster réservé aux admins (HIGH)

`GET /api/cluster/manifest` était en `require_auth` et renvoyait le **manifeste cluster brut**
(secrets `env` des services, `volumes_bind`, identifiants de transport backup) à tout
utilisateur authentifié — même classe de fuite que `/manifest/raw`, `/stacks/{name}/env` et la
vue détaillée d'app-stack, déjà fermées. L'écriture (`PUT`) était déjà admin ; la lecture passe
en **`require_admin`**.

## Backend — plus d'appels etcd bloquants dans le middleware (HIGH)

`NodeTargetMiddleware` s'exécute à **chaque requête** et résolvait le démon Docker cible
(`resolve_node_docker_host` / `resolve_container_node_docker_host`) de façon **synchrone** — en
mode cluster etcd, ce sont des allers-retours HTTP bloquants (et, pour l'auto-ciblage
conteneur, une boucle sur les nœuds). Chaque requête stallait l'event loop derrière la latence
etcd. La résolution passe désormais par `asyncio.to_thread` (comme le reste du code).

## Validation

Suite complète (747 pytest + 306 bats + vérifications shell + build frontend) au vert, puis
déploiement `2.2.19` sur le cluster 3 nœuds live (backend-only : quorum etcd 3/3, VIP flottante,
auth etcd vérifiés).
