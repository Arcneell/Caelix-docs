# Release notes — v2.2.11

**Date**: July 2026
**Image**: `ghcr.io/arcneell/caelix:2.2.11` (also `:latest`)

> Clears the audit's deferred items: unified error convention on the cluster router,
> i18n keys for the last hardcoded strings, and a widened HPA scale-down band.

---

## API — unified error convention on `cluster.py`

Every handler in `routers/cluster.py` was the last holdout from the house convention: bare
dicts with no `ok` field, and `HTTPException` raised directly in the handler body. They now
go through `json_ok` / `json_err` (`{ok:true, …}` on success, `{ok:false, error, detail}`
with the right status on error), like the rest of the API. Response fields and HTTP status
codes are unchanged (`ok` is purely additive) — the frontend, which reads fields directly,
is unaffected. The `_require_store` guard (an abort helper) keeps its `HTTPException`,
consistent with FastAPI dependencies.

## HPA — widened scale-down band

Scale-down only triggered below **0.5 × target**, leaving a very wide dead band
`[0.5·target, target]` where a service, once scaled up, would **never** scale back down even
when comfortably under target (permanent over-provisioning). The default is now
**0.8 × target** (flap-guard margin kept, replicas reclaimed across most of the band) and is
tunable per app via `hpa_down_fraction` (clamped to `[0.1, 0.95]`).

## i18n — last hardcoded strings

Routed through `t()` (fr + en keys in sync) the last untranslated user-facing strings:
App-Stack and autoscale action toasts (`{action} OK`, `Scale … (total …)`), "not found"
messages, the services count, and the container-create wizard labels (Hostname, Labels,
Capabilities, plus the Docker technical terms TTY/tini/GPUs/Sysctls/Ulimits/Tmpfs).

## Validation

Full suite (pytest + bats + shell checks + frontend build) green, then the `2.2.11` image
was deployed to the live 3-node cluster with etcd quorum, floating VIP and etcd auth
verified.
