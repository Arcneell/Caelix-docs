# Release Notes — v1.4.0

**Date**: June 2026  
**Image**: `ghcr.io/arcneell/caelix:1.4.0` (also `:latest`)

---

## Summary

v1.4.0 is the **rebranding** release: the product formerly named "SORK" becomes **Caelix** (from Latin _caelum_, the sky/cloud), with a new visual identity (the "orbit / comet" logo) and a full rename across the code, the web console, notifications, and the documentation. No feature was removed — this is a brand change, not a behaviour change.

!!! warning "Upgrading from a v1.3.x — breaking changes"
    The rebrand touches runtime identifiers. On an existing instance, the old `SORK_…` prefix becomes `CAELIX_…` and the following paths/identifiers change:

    | Item | Before | After |
    |------|--------|-------|
    | Environment variables | `SORK_*` | `CAELIX_*` |
    | Install path | `/opt/sork` | `/opt/caelix` |
    | State directory | `.sork/` | `.caelix/` |
    | Session cookie | `sork_session` | `caelix_session` (logs out current sessions) |
    | Container prefix | `sork-*` | `caelix-*` |
    | Docker label | `sork.app` | `caelix.app` |
    | systemd unit | `sork.global.service` | `caelix.global.service` |
    | Docker image | `ghcr.io/arcneell/sork` | `ghcr.io/arcneell/caelix` |

    The `etc/manifest.ini` and `etc/notify.ini` files are **never overwritten** by the installer. Migrate your environment variables to the new prefix, then run `bin/caelix once` or restart the systemd service.

---

## Changes

- **New name**: Caelix, with a dedicated wordmark and favicon.
- **New logo**: the "orbit / comet" mark (orchestrator core + comet-trail satellite), cyan→blue gradient on obsidian.
- **Full rename** preserving casing conventions: `CAELIX_*` (env), `Caelix` (UI/prose), `caelix` (paths/labels).
- **CLI**: the `bin/caelix` entrypoint replaces the former binary (same subcommands).
- **Documentation** fully re-branded (FR/EN), published at `arcneell.github.io/Caelix-docs`.
