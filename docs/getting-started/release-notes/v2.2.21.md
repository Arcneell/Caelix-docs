# Notes de release — v2.2.21

**Date** : juillet 2026
**Image** : `ghcr.io/arcneell/caelix:2.2.21` (également `:latest`)

> Deux derniers correctifs de l'audit de convergence : le YAML compose (secrets) fuyait aux
> techniciens, et `send_discord` bloquait l'event loop.

---

## Sécurité — le compose des stacks ne fuit plus aux techniciens (HIGH)

`GET /api/stacks/{name}/content` renvoyait le fichier **docker-compose brut** — avec ses blocs
`environment:` en clair (mots de passe DB, clés d'API) — à tout utilisateur authentifié.
Passé en **`require_admin`**. `GET /api/stacks/{name}/status` **masque** désormais
`compose_content` pour les non-admins (le statut des services reste visible). Même frontière
que `/state`, `manifest/raw`, la vue détaillée d'app-stack et `inspect`.

## Backend — `send_discord` sorti de l'event loop

`send_discord` exécute `curl` en sous-processus ; deux handlers (`bump_config`, `clear_pause`)
l'appelaient **directement** dans le corps async → stall de l'event loop jusqu'à ~10 s par
canal. Passés en `asyncio.to_thread`.

## Validation

Suite complète (750 pytest + 306 bats + vérifications shell + build frontend) au vert, puis
déploiement `2.2.21` sur le cluster 3 nœuds live (backend-only : quorum etcd 3/3, VIP flottante,
auth etcd vérifiés).
