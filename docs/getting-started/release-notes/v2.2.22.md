# Notes de release — v2.2.22

**Date** : juillet 2026
**Image** : `ghcr.io/arcneell/caelix:2.2.22` (également `:latest`)

> Derniers stragglers de l'audit exhaustif (80 endpoints GET énumérés) : une fuite de secrets
> de template et cinq I/O etcd bloquantes.

---

## Sécurité — corps de templates personnalisés masqués aux techniciens (HIGH)

`GET /api/templates/custom` renvoyait le **corps brut** de chaque template personnalisé —
compose/manifest pouvant embarquer `env`, `volumes_bind` et identifiants en clair — à tout
utilisateur authentifié. Les non-admins reçoivent désormais les **métadonnées** (titre,
description) mais plus le `content`. Même frontière que `/state`, `manifest/raw`, la vue
détaillée d'app-stack, `inspect` et le compose des stacks.

## Backend — cinq I/O etcd bloquantes sorties de l'event loop

Cinq handlers `async` faisaient des I/O `read_text`/`write_text`/`dir_*` synchrones (allers-
retours etcd bloquants en mode cluster) directement sur l'event loop : `get_stack_env`,
`save_stack_env`, `get_sources`, et les contrôles d'existence/suppression de
`update_custom_template` / `delete_custom_template`. Tous passés en `asyncio.to_thread`.

Avec ceci, la frontière technicien-sans-secret et la classe blocking-on-loop sont **entièrement
balayées** (audit exhaustif des ~80 endpoints GET + tous les handlers async).

## Validation

Suite complète (751 pytest + 306 bats + vérifications shell + build frontend) au vert, puis
déploiement `2.2.22` sur le cluster 3 nœuds live (backend-only).
