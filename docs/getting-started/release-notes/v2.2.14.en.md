# Release notes — v2.2.14

**Date**: July 2026
**Image**: `ghcr.io/arcneell/caelix:2.2.14` (also `:latest`)

> Robustness batch from another full audit: two **critical** fixes (daemon crash-loop on an
> invalid value, a possible double-VIP on a fenced node), plus backend, frontend and security
> hardening.

---

## Critical

- **Engine — a non-numeric `[orchestrator] interval` no longer crash-loops the daemon.**
  `interval`/`max_repair` were read unvalidated; an `interval = 15s` made `sleep` fail under
  `set -e` in the `while true` loop → **total per-node outage** in a systemd restart loop.
  Now coerced at load (and rejected by `caelix validate`).
- **Cluster — the VIP can no longer be re-bound on a fenced node.** The heartbeat loop called
  `node_vip_reconcile` without checking the fence state; on an asymmetric partition a fenced
  node (which just released the VIP) could re-bind it 5s later → double-VIP window. Now guarded
  by `node_fenced`.

## Engine — other numeric guards

Same class (non-numeric → fatal `set -u`/arith error truncating the pass):
`autoscale_up_threshold`/`down_threshold`/`cooldown`/`port_base`, `autoscale_min` (loop),
`backup_retention` — runtime coercion + `caelix validate` rejection.

## Backend

- **`/ready` no longer blocks the event loop**: the blocking DB + Docker probes run via
  `asyncio.to_thread`, so readiness polling doesn't stall other requests/SSE.
- **Aggregate `docker prune`** reflects failure: `ok` = `all(sub-prunes)` + HTTP 5xx (like the
  per-resource prune).
- **SSE slot leak fixed**: capacity is pre-checked read-only and the slot is acquired *inside*
  the generator (paired with its finally), so a never-consumed stream no longer leaks (no
  lockout after 5 connections).
- Bounds: log `tail` floored at 0, autoscale stress `amount` capped.

## Frontend

- **`sseUrl` validates the ticket** (`res.ok`/`ticket`): a missing ticket no longer produces
  `?ticket=undefined` → **no more infinite 5s SSE reconnect loop**; capped exponential backoff
  (5s→60s) reset on open.
- **`LogsView`**: the expanded row is tracked by a stable identity (ts+message) instead of the
  array index — the 5s poll no longer expands the wrong line.
- **`SettingsView`**: source rows keyed by identity (`_id`) — no more wrong field rebind when a
  row is deleted.

## Security

- **`svc_image`** (app-stack deploy + add-service) validated with `_valid_docker_name`
  (defense in depth: prevents an image shaped like a `--…` option being parsed as a flag,
  consistent with the rest of the codebase).

## Validation

Full suite (741 pytest + 303 bats + shell checks + frontend build) green, then the `2.2.14`
image was deployed to the live 3-node cluster with etcd quorum, floating VIP and etcd auth
verified.
