# Release notes — v2.2.10

**Date**: July 2026
**Image**: `ghcr.io/arcneell/caelix:2.2.10` (also `:latest`)

> Robustness batch from a full audit: two guards that stop an invalid manifest value from
> wedging reconciliation, bcrypt moved off the event loop, the container-detail view finally
> node-aware, and leadership/liveness hardening.

---

## Robustness — engine (reconcile)

Two non-numeric manifest values reached an arithmetic context and raised a **fatal**
`set -u` error, uncatchable by the callers' `|| return 0` — which **killed the entire
reconcile pass** (for every app), every cycle, until fixed by hand:

- a `backup_schedule` that is neither an alias (`hourly`/`daily`/`weekly`) nor an integer
  (e.g. `every-night`) → `_backup_due`.
- a non-numeric `health_max_bytes` (e.g. `1M`) → `health_http`.

Both are now **coerced to a safe default** at runtime (daily / disabled) and **rejected at
load time** by `caelix validate` (defense in depth). bats tests added.

## Security / availability — backend

- **bcrypt off the event loop**: `login`, `me` and `change-password` ran the bcrypt verify
  (~250-400ms, GIL) **synchronously** inside an `async` handler, freezing the whole console
  (SSE, polls, other requests) for the hash duration — a cheap DoS lever since the
  dummy-hash path always hashes, even for a non-existent login. Moved to
  `await asyncio.to_thread(verify_password, …)`.
- **HTTP status fixes**: a failed Docker prune (`docker_prune_single`) and a failed legacy
  container action returned **200**; an unknown app returned **500**. They now return **5xx**
  and **404** respectively. Unguarded `int()` (SMTP port, `Content-Length` on restore/import)
  now return a clean **400** instead of a raw 500.

## Robustness — cluster (HA)

- **`ContainerDetailView` is node-aware** *(frontend)*: the detail view ignored the `?node=`
  the list passes → in a cluster, inspect/logs/stats/exec/rename/start/stop/remove and both
  SSE streams all hit the **controller** daemon, showing "not found" or acting on the wrong
  host for any container on a worker. Every call and SSE now targets the container's node.
- **Leadership bound to the lease**: `acquire_leadership` re-acquires by value only if the
  key is **bound to the current session** (the value-only check let a key from a previous
  term feign leadership under a dying lease → double-VIP window). On resign, the loop now
  resets `session=""` so a revoked lease is never reused.
- **Liveness — bounded future skew**: a heartbeat far in the future (skewed clock) is no
  longer treated as alive forever (beyond one TTL ahead = suspect), otherwise a dead,
  mis-clocked node blocked rescheduling and VIP failover.

## Validation

Full suite (pytest + bats + shell checks + frontend build) green, then the `2.2.10` image
was deployed to the live 3-node cluster with etcd quorum, floating VIP and etcd auth
verified.
