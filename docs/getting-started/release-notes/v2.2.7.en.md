# Release notes — v2.2.7

**Date**: July 2026
**Image**: `ghcr.io/arcneell/caelix:2.2.7` (also `:latest`)

> Typed request models (Pydantic) on **every** API write endpoint. No behaviour change —
> request bodies are now documented and validated at the boundary.

---

## Typed request bodies

The 64 endpoints that took an untyped `body: dict` now use a Pydantic model
(`app/models.Body`) declaring their well-known fields. Benefits:

- **OpenAPI docs**: every request body appears with its schema in `/docs` (60 models
  generated) — previously an opaque arbitrary object.
- **Typed validation** at the boundary (declared fields are validated/coerced).
- Less defensive `str(body.get(...))` over time.

### Risk-free migration (behaviour preserved)

The pattern is designed to be **strictly equivalent** to the old `body: dict`:

- `extra="allow"` base → any extra field the frontend sends is **kept**.
- Handlers do `body = payload.model_dump(exclude_unset=True)` → `body` is exactly the
  dict the client sent (only *sent* fields present), so every `body.get(key, default)`
  and `if key in body` keeps its behaviour.
- Only **declared** fields are typed — adoptable field by field without touching handler
  logic.
- Optional-body endpoints keep a default (`payload: X = X()`).

## Validation

Full backend suite green (**716 + pattern**) — every endpoint is exercised with real
payloads (+ extra fields). Console rebuilt and served on the 3-node cluster; OpenAPI
schema verified (`/docs`, `/openapi.json`).
