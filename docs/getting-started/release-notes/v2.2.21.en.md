# Release notes — v2.2.21

**Date**: July 2026
**Image**: `ghcr.io/arcneell/caelix:2.2.21` (also `:latest`)

> The last two convergence-audit fixes: the compose YAML (secrets) leaked to technicians, and
> `send_discord` blocked the event loop.

---

## Security — stack compose no longer leaks to technicians (HIGH)

`GET /api/stacks/{name}/content` returned the **raw docker-compose** file — with its plaintext
`environment:` blocks (DB passwords, API keys) — to any authenticated user. Now
**`require_admin`**. `GET /api/stacks/{name}/status` now **redacts** `compose_content` for
non-admins (service status stays visible). Same boundary as `/state`, `manifest/raw`, the
app-stack detail view and `inspect`.

## Backend — `send_discord` off the event loop

`send_discord` runs `curl` as a subprocess; two handlers (`bump_config`, `clear_pause`) called
it **directly** in the async body → event-loop stall of up to ~10s per channel. Moved to
`asyncio.to_thread`.

## Validation

Full suite (750 pytest + 306 bats + shell checks + frontend build) green, then `2.2.21`
deployed to the live 3-node cluster (backend-only: etcd 3/3 quorum, floating VIP, etcd auth
verified).
