# Notes de release — v1.4.0

**Date** : juin 2026  
**Image** : `ghcr.io/arcneell/caelix:1.4.0` (également `:latest`)

---

## Résumé

La v1.4.0 est la release de **rebranding** : le produit anciennement nommé « SORK » devient **Caelix** (de _caelum_, le ciel/cloud), avec une nouvelle identité visuelle (logo « orbite / comète ») et le renommage complet du code, de la console web, des notifications et de la documentation. Aucune fonctionnalité retirée — c'est un changement de marque, pas de comportement.

!!! warning "Mise à jour depuis une v1.3.x — changements cassants"
    Le rebranding touche des identifiants runtime. Sur une instance existante, l'ancien préfixe `SORK_…` devient `CAELIX_…` et les chemins/identifiants suivants changent :

    | Élément | Avant | Après |
    |---------|-------|-------|
    | Variables d'environnement | `SORK_*` | `CAELIX_*` |
    | Chemin d'installation | `/opt/sork` | `/opt/caelix` |
    | Dossier d'état | `.sork/` | `.caelix/` |
    | Cookie de session | `sork_session` | `caelix_session` (déconnecte les sessions en cours) |
    | Préfixe des conteneurs | `sork-*` | `caelix-*` |
    | Label Docker | `sork.app` | `caelix.app` |
    | Unité systemd | `sork.global.service` | `caelix.global.service` |
    | Image Docker | `ghcr.io/arcneell/sork` | `ghcr.io/arcneell/caelix` |

    Les fichiers `etc/manifest.ini` et `etc/notify.ini` ne sont **jamais écrasés** par l'installeur. Reportez vos variables d'environnement sur le nouveau préfixe, puis relancez `bin/caelix once` ou redémarrez le service systemd.

---

## Changements

- **Nouveau nom** : Caelix, avec wordmark et favicon dédiés.
- **Nouveau logo** : mark « orbite / comète » (noyau orchestrateur + satellite à traînée), dégradé cyan→bleu sur fond obsidienne.
- **Renommage complet** en préservant les conventions de casse : `CAELIX_*` (env), `Caelix` (UI/prose), `caelix` (chemins/labels).
- **CLI** : l'entrée `bin/caelix` remplace l'ancien binaire (mêmes sous-commandes).
- **Documentation** FR/EN entièrement remarquée, publiée sur `arcneell.github.io/Caelix-docs`.
