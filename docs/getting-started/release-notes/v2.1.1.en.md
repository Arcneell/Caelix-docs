# Release notes — v2.1.1

**Date**: June 2026  
**Image**: `ghcr.io/arcneell/caelix:2.1.1` (also `:latest`)

> An interface-polish release: clearer list loading, instant navigation via a cache,
> restrained animations, and several dark-mode display fixes. No engine, API, or
> configuration changes — a drop-in update from 2.1.0.

---

## Loading and navigation

- **Resource lists (Containers, Images, Networks, Volumes)** now wait for **every
  node** to answer and then reveal the list **all at once**, behind a loading
  skeleton — no more rows popping in node by node.
- **Stale-while-revalidate cache**: returning to a page you've already opened paints
  the last-known list **instantly**, then revalidates in the background. The only
  network wait happens on the very first visit of the session.

## Animations

- Page transitions (a short fade on navigation), Overview counters that tween to
  their value and pulse when they change, and shimmer skeletons during loads.
- Every animation honours the system `prefers-reduced-motion` preference and
  collapses to near-instant for motion-sensitive users.

## Display fixes

- **Dark mode**: text inside modals (wizards, dialogs) that rendered as unreadable
  black is fixed — the base colour is now set on `body`, so any overlaid content
  inherits a readable colour.
- **Header menus**: the dropdowns (notifications, user menu) rendered behind the page
  content; they now float above it.
- **User menu**: "Change password" no longer overflows or stacks in French — the menu
  sizes to its content.

---

!!! note "Upgrade"
    Update the image to `ghcr.io/arcneell/caelix:2.1.1` and recreate the console
    container on each node. No data or configuration migration is required from 2.1.0.
