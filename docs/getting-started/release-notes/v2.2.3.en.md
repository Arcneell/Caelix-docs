# Release notes — v2.2.3

**Date**: July 2026
**Image**: `ghcr.io/arcneell/caelix:2.2.3` (also `:latest`)

> **Quality, correctness and robustness** fixes from a global code audit (beyond
> security). No visible functional change.

---

## Backend (correctness / performance)

- **Non-blocking cluster endpoints**: `cluster.py` handlers (and `metrics`, `logs`,
  `incidents`, `registry`) ran synchronous I/O (etcd, docker, files) **on the event
  loop**, stalling every request and SSE stream. They now run in the threadpool.
- **Limiter memory leaks**: the brute-force counters (rate-limit, login lockout) never
  released aged-out keys → unbounded growth under IP rotation. Added opportunistic
  sweeping + empty-key deletion.
- **Write races**: `certificates.json` (renewal vs upload) under a shared lock;
  incident-ack serialized; HPA applies its changes via a re-read merge instead of
  clobbering the manifest; state cache locked.
- **Robustness**: a corrupt cluster user store now yields a controlled error (no more
  500 on every login); notification history not lost on a transient read error;
  notification-delivery failure logged instead of swallowed; `registry` tolerates an
  entry missing `name`.

## Cluster / HA

- **HPA autoscaler** now measures **all** replicas (`-rN`), not just the first → no
  more premature scale-down / oscillation; counters pruned for deleted apps.
- **Resignation hysteresis**: the controller no longer hands off leadership (and the
  VIP) on a single missed heartbeat — it requires 2 consecutive ticks (anti-flapping).

## Shell engine

- **A corrupt timestamp file no longer kills the reconcile pass** (numeric guard before
  arithmetic under `set -u`, in backup and update).
- **DB backup** checks the dump's exit code (`PIPESTATUS`) and gzip integrity
  (`gzip -t`) — a truncated dump is no longer recorded as successful (and is removed).
- **Proxy**: non-numeric `Content-Length` and invalid `--health-interval` handled (no
  more broken handler or 100% CPU busy-spin); etcd leader id trimmed (reliable VIP
  bind); autoscale metric validated before comparison.

## Frontend / build

- Cap the live-log buffer (memory leak on a long stream).
- Release image built on **Node 20 (LTS)** — the toolchain CI validates (was Node 26,
  never exercised by CI).

## Tests

- At-rest encryption exercised for real (seal/open through `shared_state`, migration);
  etcd 401-retry test; configurable bcrypt cost (suite ~2× faster in CI).

---

> systematic `reqId` anti-race guard across all frontend views.
