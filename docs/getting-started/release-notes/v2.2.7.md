# Notes de release — v2.2.7

**Date** : juillet 2026
**Image** : `ghcr.io/arcneell/caelix:2.2.7` (également `:latest`)

> Modèles de requête typés (Pydantic) sur **tous** les endpoints d'écriture de l'API.
> Aucun changement de comportement — les corps de requête sont désormais documentés et
> validés à la frontière.

---

## Corps de requête typés

Les 64 endpoints qui recevaient un `body: dict` non typé utilisent maintenant un modèle
Pydantic (`app/models.Body`) déclarant leurs champs connus. Gains :

- **Documentation OpenAPI** : chaque corps de requête apparaît avec son schéma dans
  `/docs` (60 modèles générés) — avant, un objet arbitraire opaque.
- **Validation typée** à la frontière (les champs déclarés sont validés/coercés).
- Moins de `str(body.get(...))` défensif à terme.

### Migration sans risque (comportement préservé)

Le pattern est conçu pour être **strictement équivalent** à l'ancien `body: dict` :

- Base `extra="allow"` → tout champ supplémentaire envoyé par le frontend est **conservé**.
- Handlers : `body = payload.model_dump(exclude_unset=True)` → `body` redevient
  **exactement** le dict envoyé par le client (seuls les champs *envoyés* sont présents),
  donc chaque `body.get(clé, défaut)` et `if clé in body` garde son comportement.
- Seuls les champs **déclarés** sont typés — adoptables champ par champ sans toucher la
  logique des handlers.
- Les endpoints à corps optionnel gardent un défaut (`payload: X = X()`).

## Validation

Suite backend complète verte (**716 + pattern**) — chaque endpoint est exercé avec de
vrais payloads (+ champs extra). Console re-buildée et servie sur le cluster 3 nœuds ;
schéma OpenAPI vérifié (`/docs`, `/openapi.json`).
