# Release notes — v2.1.0

**Date**: June 2026  
**Image**: `ghcr.io/arcneell/caelix:2.1.0` (also `:latest`)

> 2.1 replaces the cluster store with **etcd (Apache-2.0)** and removes Consul
> entirely, protects the infrastructure containers from the console, and modernises
> the interface (gradient theme, loading skeletons). No service-configuration
> breakage; single-host is unchanged.

---

## Cluster store: etcd (Apache-2.0), Consul removed

- The HA control plane now runs on **etcd**: every node runs an etcd member (Raft
  quorum) and a controller, leader election uses a **lease + a put-if-absent
  transaction**, and shared state (users, JWT secret, configuration, stacks,
  certificates) lives in etcd. The client API is on `:2379`, raft on `:2380`.
- **Consul is removed entirely** from the product (code, options, documentation).
  The installer offers only the etcd store; a node joins with
  `--mode join --store-addr http://<existing-node>:2379` (joining the etcd cluster is
  automatic via `etcdctl member add`).
- For commercial distribution this means **no BSL-licensed dependency** remains; a
  standard deployment ships only permissively licensed components.

!!! note "Migrating from 2.0"
    The store changes from Consul to etcd. On each node: update the image and
    re-extract the engine, remove the old Consul container (`docker rm -f
    caelix-consul` and its volume), then reinstall in cluster mode with
    `--store-addr`. Controller first, the other nodes after.

## Infrastructure container protection

The console now marks its infrastructure containers — the web console
(`caelix-caelix-ui`) and the store (`caelix-etcd`) — as **system**. They are
**hidden by default** in the Containers view (a "System" toggle reveals them),
flagged with a badge, and any stop or delete goes through an **explicit warning**:
the operation breaks the node and can break the cluster quorum.

## Interface

- **Gradient theme** applied across the console in dark and light variants, with a
  translucent sidebar and header. The light theme is no longer plain white.
- **Loading skeletons**: resource lists (containers, images, volumes, networks,
  services, stacks…) show an animated placeholder while loading instead of popping
  in all at once.

---

## Upgrade

```bash
docker run --rm ghcr.io/arcneell/caelix:latest cat /opt/caelix/install.sh | bash -s -- --with-systemd
```

On a cluster, update each node (image + engine re-extraction); see the migration
note above for moving from Consul to etcd.
