# Notes de release — v2.2.15

**Date** : juillet 2026
**Image** : `ghcr.io/arcneell/caelix:2.2.15` (également `:latest`)

> Lot issu de la campagne d'audit continue : fermeture d'une **fuite de secrets** vers le
> rôle technicien, garde anti-crash sur `auto_update_interval`, fin des OOM sur les
> exports/backups volumineux, et fiabilisation frontend/cluster.

---

## Sécurité — fuite de secrets aux techniciens (HIGH)

Trois lectures renvoyaient le manifeste/`.env` **en clair** (mots de passe backup FTP/rsync,
blocs `env` des services) à tout utilisateur authentifié, alors que `/state` les masque déjà
pour les non-admins et que leurs écritures sont admin-only :
`GET /api/manifest/show`, `GET /api/manifest/raw`, `GET /api/stacks/{name}/env`.
Elles sont désormais **`require_admin`**.

## Moteur — garde anti-crash + robustesse

- **`auto_update_interval`** non numérique (ex. `nightly`) atteignait `(( ))` → erreur `set -u`
  fatale abrégeant la passe de réconciliation. Coercition runtime + rejet par `caelix validate`
  (alias `hourly`/`daily`/`weekly` toujours acceptés).
- **Creds etcd échappés** dans le corps JSON d'authentification (`lib/node.sh`) : un mot de
  passe contenant `"`/`\` ne casse plus le handshake.

## Backend — fin des OOM sur gros transferts

`docker save` (export d'image), `docker export` (export de conteneur) et le backup de
configuration bufferisaient l'archive **entière en RAM** → risque d'OOM de la console sur une
image/un conteneur de plusieurs Go. Ils écrivent maintenant dans un fichier temporaire et le
**streament depuis le disque** (`FileResponse`, nettoyé après envoi). Bornes ajoutées sur
`limit` (`/incidents`, `/audit`) ; deux `except (X, Exception)` fourre-tout resserrés.

## Frontend — clés stables (fin des mauvais rebinds)

Les listes d'édition supprimables par `splice` utilisaient la clé d'index → à la suppression
d'une ligne, les valeurs/le focus des champs sautaient sur la mauvaise ligne. Passées à une
**clé d'identité stable** (`_id`) : CreateContainerView (ports/volumes/env/labels + listes
capabilities/devices/ulimits/dns/hosts/tmpfs/log-opts/sysctls), ServicesView et StackDetailView
(variables d'env). Libellés de ContainerDetailView passés par `t()`.

## Cluster (HA)

- **`renew_session` réessaie une fois** le keepalive avant de déclarer le bail perdu : un 5xx
  etcd transitoire ne provoque plus la révocation d'un bail encore valide (churn de leadership /
  flap VIP sur erreur passagère). Une vraie partition échoue toujours après le retry.

## Validation

Suite complète (744 pytest + 304 bats + vérifications shell + build frontend) au vert, puis
déploiement de l'image `2.2.15` sur le cluster 3 nœuds live avec vérification du quorum etcd,
de la VIP flottante et de l'authentification etcd.
