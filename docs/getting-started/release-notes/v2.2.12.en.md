# Release notes — v2.2.12

**Date**: July 2026
**Image**: `ghcr.io/arcneell/caelix:2.2.12` (also `:latest`)

> Robustness batch from another full audit: more reconcile anti-abort guards, two backend
> event-loop/status fixes, HA hardening, and a git-deploy RCE guard.

---

## Engine — reconcile can no longer be wedged by an invalid value

Following the 2.2.10 guards, more manifest numerics reached an arithmetic context and
raised a **fatal** `set -u` error (uncatchable), which **truncated the whole reconcile
pass**: `autoscale_min`, `autoscale_max` (`autoscale.sh`) and `memory_soft_mb` /
`memory_hard_mb` (`health.sh`). They are now coerced to a safe default at runtime **and**
rejected at load time by `caelix validate`. bats tests (`validate.bats`).

## Backend

- **`container_exec`** always returned `ok:true` / HTTP 200, even for a stopped/missing
  container. `ok` now reflects whether `docker exec` itself succeeded (failure → **5xx**);
  a non-zero command exit is still a valid exec (200, surfaced via `exit_code`).
- **Notifications endpoints** (`list`/`read`/`delete`) are now **synchronous** handlers
  (threadpool) since they do file I/O — as `async def` they blocked the event loop on every
  SPA poll (same class as the bcrypt fix).
- **Availability guard** on `PUT /users`: an admin can no longer deactivate themselves or
  remove/demote the **last active admin** (prevents a total lockout).

## Cluster (HA)

- **etcd `_post` catches socket timeouts** (`OSError`/`TimeoutError`, not just `URLError`):
  on a partial partition (connect ok, keepalive read stalls) the exception no longer
  escapes — `renew_session` returns `False` and the controller **resigns** instead of
  holding a dead lease (stale-VIP / split-brain window).
- **Liveness — relaxed future-skew bound**: realistic NTP drift (up to 300s) no longer
  flips a **live** node to dead (avoiding needless eviction / VIP failover); only an
  implausibly far-future timestamp is still rejected.
- **HPA**: the delta is applied only if `total_replicas` hasn't changed since the read (no
  longer fights a concurrent manual scale); the `hpa_down_fraction` cap drops to 0.85
  (anti-oscillation margin).

## Security

- **`git clone` RCE closed** (`/stacks/deploy-git`, admin): `repo_url` is restricted to a
  real network transport (http(s)/git/ssh) — git's local transport helpers (`ext::sh -c …`,
  `fd::`, `file://`) and a leading `-` are rejected; `GIT_ALLOW_PROTOCOL` and
  `GIT_TERMINAL_PROMPT=0` as defense in depth; leading-`-` guard on `branch`.
- **certbot**: the `email` (`-m` argument) is now validated (email shape, no leading `-`).

## Frontend

- `useRequestGuard` extended to the last unguarded fetches (SystemView, EventsView,
  IncidentsView, LogsView `fetchStats`) — no more node-switch/refresh races.
- `JournalView`: `String()` crash-guard on a non-string detail.
- `ContainersView`: action verb and Start/Stop/Restart titles via `t()` (i18n).

## Validation

Full suite (735 pytest + 292 bats + shell checks + frontend build) green, then the
`2.2.12` image was deployed to the live 3-node cluster with etcd quorum, floating VIP and
etcd auth verified.
