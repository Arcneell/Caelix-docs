# Release notes — v2.1.3

**Date**: June 2026  
**Image**: `ghcr.io/arcneell/caelix:2.1.3` (also `:latest`)

> An interface fix: the dashboard shows recent incidents again. No engine, API, or
> configuration changes — a drop-in update from 2.1.2.

---

## Fixes

- **Dashboard — "Recent incidents"** was always empty. The API returns incidents as
  raw log lines (`timestamp | severity | service | event | detail`); the dashboard
  treated each as an object, so every field came out blank. The lines are now parsed
  correctly, shown **most recent first**, with the timestamp formatted in the user's
  timezone.

---

!!! note "Upgrade"
    Update the image to `ghcr.io/arcneell/caelix:2.1.3` and recreate the console
    container on each node. No data or configuration migration is required from 2.1.2.
